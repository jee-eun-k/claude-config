# Sync Sprint Command

Reads `sprint-status.yaml` and generates an Obsidian sprint dashboard.

## Data Sources

- **Sprint status**: `~/Development/_bmad-output/brs-client/sprint-status.yaml`
- **Obsidian dashboard**: `~/Library/Mobile Documents/com~apple~CloudDocs/000_obsidian/000_work/dev-notes/sprint-dashboard.md`

## Execution Steps

### Step 1: Load and Parse Sprint Status

Read `sprint-status.yaml` and parse the `development_status` section. Classify each entry:
- **Epic**: key starts with `epic-` and does NOT end with `-retrospective`
- **Retrospective**: key ends with `-retrospective`
- **Story**: everything else

Extract for each story:
- `story_id`: the YAML key (e.g., `5-2d-fix-unknown-proper-types`)
- `status`: the value (e.g., `backlog`, `done`)
- `epic`: derived from the story ID prefix (e.g., `5` → `epic-5`)
- `display_title`: humanize the ID — strip numeric prefix, convert kebab-case to Title Case, prepend bracket prefix
  - Example: `5-2d-fix-unknown-proper-types` → `[5.2d] Fix Unknown Proper Types`
  - Example: `1-1-make-pr-checks-blocking` → `[1.1] Make PR Checks Blocking`

Also extract epic metadata:
- `epic_id`: the key (e.g., `epic-5`)
- `epic_status`: the value
- `epic_title`: from the YAML comment above it (e.g., `# Epic 5: Remove Build Bypasses (Phase 5)`)

### Step 2: Generate Obsidian Sprint Dashboard

Write `sprint-dashboard.md` to the Obsidian vault with this structure:

```markdown
---
title: BMAD Sprint Dashboard
project: <project name from sprint-status>
project_key: <project_key>
generated: <current ISO timestamp>
tags: [bmad, sprint, auto-generated]
---

# BMAD Sprint Dashboard
> Auto-generated by `/sync-sprint`. Do not edit manually.
> Last synced: <timestamp>

## Progress Overview

| Metric | Count |
|--------|-------|
| Done | <n> |
| In Progress | <n> |
| Review | <n> |
| Ready for Dev | <n> |
| Backlog | <n> |
| **Total** | **<n>** |

Progress: <unicode progress bar> <percent>%

## <Epic Title> (<epic status>)

| Story | Status |
|-------|--------|
| <display_title> | <status badge> |

<!-- Repeat per epic, active epics first, then completed in collapsed details -->

<details><summary>Completed Epics</summary>

## <Completed Epic Title> (done)
| Story | Status |
|-------|--------|
| <display_title> | done |

</details>
```

**Status badges** (plain text, Obsidian-compatible):
- `backlog` → `backlog`
- `ready-for-dev` → `ready`
- `in-progress` → `**IN PROGRESS**`
- `review` → `*review*`
- `done` → `~~done~~`

### Step 3: Report Summary

Print a summary:
- Stories by status (done/in-progress/review/backlog counts)
- Overall progress percentage
- Obsidian dashboard path
- Any errors encountered
